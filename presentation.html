Part 1 · Concept

What is a WAF?

  • Sits in front of the web app — first line of defense.
  • Works at the application layer (HTTP/HTTPS).
  • Inspects each request — URL, headers, body — and blocks malicious ones.
  • Catches what a traditional network firewall can't see: SQLi, XSS, and friends.
👤 User / Attacker

🛡️ DP WAF — inspect & filter

🖥️ Origin web app
⏱ ~2 min
Keep short. Network firewall = L3/4 (IP/port); WAF = L7, reads the HTTP payload. Analogy: door lock vs. guard reading the envelope.

Part 1 · Attack 1 live demo

SQL Injection — login bypass

The login builds its query by gluing your input into SQL text:

SELECT * FROM users
WHERE user = '' OR 1=1 --' AND pass = ''

Payload ' OR 1=1 -- (the lab's click-to-fill chip) makes the query return every row — -- comments out the password check. Alt: admin' -- skips straight to the admin row.

Demo steps

  1. Go to /login.html
  2. Username: ' OR 1=1 -- (chip) · password blank → Sign in
  3. Show the returned executedSql
⏱ ~3 min
DEMO: 1) login guest/guest (works) 2) login ' OR 1=1 -- + blank password → admin 3) point at executedSql panel. Real fix = parameterized queries; edge fix = DP WAF.

Part 2 · Demo — Phase 1 live demo

One origin hits the wall

With only one VPS origin active, that origin's 2-IP capacity limit is the whole ceiling.

Everyone opens /capacity.html from their own device. The 3rd person sees:

HTTP 503

A real overload — their own IP, rejected. Single origin = single capacity ceiling.

Part 2 · Concept

DDoS protection — per-IP awareness

  • The presenter runs a flood script from their own IP — a burst of requests at DP WAF.
  • DP WAF DDoS protection detects the abnormal volume and blocks that IP.
  • The presenter opens the site in their browser → can no longer access it — DP WAF blocked their real IP.
  • Against the real DP WAF, the flood comes from the presenter's real client IP — it ignores fake X-Forwarded-For headers, so the block is genuine, not a simulation.

Per-IP awareness, applied defensively: spread the good, drop the bad — at the edge.

⏱ ~2 min
DEMO: run the flood script from your own IP. DP WAF detects the volume and blocks you. Open the site in your browser → it's now inaccessible to you. Key point: real client IP, not a spoofed header — the block is real. NOC sees: spike from one IP → flatlines once blocked.
Wrap-up

Takeaways for NOC

  • 🛡️ WAF = layer-7 guard. Blocks SQLi & XSS — injections the network firewall can't see.
  • ⚖️ Load balancing = availability. Horizontal scale removes the single-origin wall.
  • 🚧 Per-IP control = DDoS defense. Spread the good, drop the bad, at the edge.
  • 👀 You're the first responders. Now you know the signatures: 503 spikes, single-IP floods, injection strings in logs.
⏱ ~2 min
One-liner: the edge (WAF + LB) enforces both security + availability before the origin. Encourage replaying the lab. Invite questions.
